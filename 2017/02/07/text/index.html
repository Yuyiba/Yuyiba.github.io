从Hello World到Hello Kitty的转变

发表于 2017-02-07

干这一行的没有自己的github和blog怎么能行，将来在自己的简历上最下面用微软雅黑写上www.github.com/yuyiba和www.beautfer.com岂不是看起来很棒棒，要举高高的样子。很多人可能会问，这和知乎，简书，微博，有什么区别，其他的平台页面还更好看，功能更齐全呢！唉 谁知道呢？可能是因为我们很闲吧=3=

blog零几年刚开始流行的时候，我们都还是一群毛没长齐，嘴里嚼着大大的小屁孩，没有机会接触到。现在长大了，时代进步了，马斯克的火箭都能直直落地了，blog的发展情况却愈发颓靡，很少人会在这里写文章，讲故事，只有一小撮热爱blog的人和程序员还在仍然这里记录分享着自己的学习记录或思想感悟。

比如这几天用github pages和hexo搭建这个博客，搭建之前看网上都有教程，心想着不难就一步步做下去，恩……一步步做了三天。完成之后我总结了为什么一天可以做完的事情我TM花了三天时间，甚至到最后我盯着屏幕上一个个没有错误的标点符号想着我是不是需要杨老师用他的宝贝电一电。

搭建过程中，我用的一篇教程名为“史上最详细的XXX教程”，然后……没成功，我就又找了一篇“史上最强的XXXX教程”还不成功，后来看了无数篇“史上最XXXX的教程”实践失败后，我发现这些教程都挺相似的，不是内容的相似，是辣鸡程度都挺相似的，基本都是把别人的教程拿过来增删改查，这并不可怕，可怕的是他们都不实践一下就发出去，很多步骤一旦有问题，后面的代码哪怕再合适也不行，搞得我后来打开了十多个教程研究，最终用自己单身20年的双手解救了自己，真是可怜了我这样的菜鸟。当然其中不乏特别好的教程，要不然怎可能这么“轻松”的完成。
夜深了……
最后……
愿在此能撇弃其他平台的浮躁与虚无，做到如nexT主题所说的“精于心，简于型”，在这一片自留地努力记录技术方面内容与想法，既是自己的学习总结，也可作他人交流使用，共同成长。
